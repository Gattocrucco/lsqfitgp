40  

41 Base class to represent kernels, i.e., covariance functions. 

42 

43 A kernel is a two-argument function that computes the covariance between 

44 two functions at some points according to some probability distribution: 

45 

46 .. math:: 

47 \mathrm{kernel}(x, y) = \mathrm{Cov}[f(x), g(y)]. 

48  

49 `CrossKernel` objects are callable, the signature is ``obj(x, y)``, and 

50 they can be summed and multiplied between them and with scalars. They 

51 are immutable; all operations return new objects. 

52 

53 Parameters 

54 ---------- 

55 core : callable 

56 A function with signature ``core(x, y)``, where ``x`` and ``y`` 

57 are two broadcastable numpy arrays, which computes the value of the 

58 kernel. 

59 derivable, scale, loc, maxdim, dim : 

60 If specified, these arguments are passed as arguments to the 

61 correspondingly named transformations, in the order listed here. See 

62 `transf.` Briefly: the kernel selects only fields `dim` in the 

63 input, checks the dimensionality against `maxdim`, transforms as 

64 ``(x - loc) / scale``, then sets the degree of differentiability. If 

65 any argument is callable, it is passed `**kw` and must return the 

66 actual argument. 

67 forcekron : bool, default False 

68 If True, apply `Kernel.forcekron` before the other transformations. 

69 batchbytes : number, optional 

70 If specified, apply ``.batch(batchbytes)`` to the kernel. 

71 **kw 

72 Additional keyword arguments are passed to `core` and saved as 

73 `initargs` attribute. 

74  

75 Attributes 

76 ---------- 

77 derivable : pair of int or None 

78 How many times each function is (mean-square sense) derivable. 

79 ``sys.maxsize`` if it is smooth. `None` mean unknown. 

80 initargs : dict 

81 The `kw` argument. Propagates when the kernel is transformed. 

82  

83 Methods
